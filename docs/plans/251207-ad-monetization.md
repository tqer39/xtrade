# 無料版の広告表示による収益化計画

- **作成日**: 2025-12-07
- **ステータス**: Phase 1-2 実装完了
- **担当 Agent**: UIAgent, ArchAgent

## 概要

無料版ユーザーに対して広告を表示し、LLM 使用料金および SaaS 費用をカバーする収益を得る。

## 調査結果

### 1. Web 広告サービス比較

| サービス | 特徴 | 審査難易度 | 最低支払い | 日本対応 | Next.js 対応 |
| --- | --- | --- | --- | --- | --- |
| **Google AdSense** | 最大手、CPM 型に変更（2024〜）、高信頼性 | 高 | $100（約15,000円） | ◎ | ◎ |
| **i-mobile** | 国内大手、審査容易、Google 認定パートナー | 低 | 3,000円 | ◎ | ○ |
| **Media.net** | Yahoo! ネットワーク、コンテキスト広告 | 中 | $100 | ○ | ○ |
| **PropellerAds** | 審査容易、多様な広告形式 | 低 | $5 | ○ | ○ |

#### 推奨: Google AdSense

**理由:**

1. Next.js との統合が成熟している（`next/script` + 専用ライブラリ）
2. 2024年の CPM 型移行により、クリック依存度が低下
3. トレカ取引サイトのユーザー層と広告主のマッチングが良い
4. 長期的な収益安定性と信頼性

**代替案:** 審査落ち時は i-mobile を第二候補として検討。

### 2. 広告表示戦略

#### UX を損なわない広告配置

```text
┌─────────────────────────────────────┐
│  ヘッダー（広告なし）                │
├─────────────────────────────────────┤
│  メインコンテンツ                    │
│  ・カード検索結果                    │
│  ・取引一覧                          │
├─────────────────────────────────────┤
│  【広告枠 A】検索結果下部（任意）     │ ← 主要コンテンツの邪魔にならない
├─────────────────────────────────────┤
│  追加コンテンツ                      │
├─────────────────────────────────────┤
│  【広告枠 B】フッター上部             │ ← 離脱ポイント手前
├─────────────────────────────────────┤
│  フッター                            │
└─────────────────────────────────────┘
```

#### 表示ルール

| ルール | 理由 |
| --- | --- |
| 取引フロー中は非表示 | 離脱防止・誤タップ防止 |
| モーダル/ダイアログ内は非表示 | 操作妨害防止 |
| 1 ページ最大 2 枠 | 過剰広告による離脱防止 |
| ファーストビュー外に配置 | 主要コンテンツへのアクセス優先 |
| ログインユーザーには控えめに | 将来の課金ユーザー候補 |

#### 広告形式

- **レスポンシブディスプレイ広告**: 自動でサイズ調整、レイアウト崩れ防止
- **インフィード広告は避ける**: カード一覧と混同されやすい

### 3. 収益性試算

#### 現実的な単価目安（日本市場 2024-2025）

| 指標 | 目安 |
| --- | --- |
| CPM（1,000 インプレッション単価） | 50〜200 円 |
| RPM（1,000 PV あたり収益） | 100〜400 円 |
| 特化サイトのRPM | 500〜1,000 円 |

#### 月間コスト見積もり

| 項目 | 月額コスト | 備考 |
| --- | --- | --- |
| Vercel | $0〜20 | 無料枠で十分な可能性 |
| Neon | $0〜25 | 無料枠あり、超過時は従量課金 |
| Cloudflare | $0 | 無料プランで十分 |
| ドメイン | 〜$2 | 年額按分 |
| **小計（SaaS）** | **$0〜50（0〜7,500 円）** | |
| LLM（将来） | $10〜100+ | 使用量次第 |
| **合計** | **$10〜150（1,500〜22,500 円）** | |

#### 損益分岐点の試算

**前提条件:**

- 月間コスト: 5,000 円（SaaS のみ）〜 20,000 円（LLM 込み）
- RPM: 200 円（控えめな見積もり）

| シナリオ | 必要月間 PV | 現実性 |
| --- | --- | --- |
| 5,000 円/月 カバー | 25,000 PV | △ 中期目標として妥当 |
| 10,000 円/月 カバー | 50,000 PV | × 相当なユーザー数が必要 |
| 20,000 円/月 カバー | 100,000 PV | × 大規模サービス水準 |

### 4. 実現の可能性評価

#### AdSense 審査の課題

1. **必要なページ**: プライバシーポリシー、利用規約、お問い合わせ
2. **コンテンツ量**: 10〜15 ページ以上の有用なコンテンツ
3. **審査期間**: 1〜2 週間
4. **リスク**: 取引サイトは審査に落ちやすいジャンル

#### 結論

| 評価項目 | 判定 | 理由 |
| --- | --- | --- |
| 技術的実現性 | ◎ | Next.js との統合は容易 |
| AdSense 審査通過 | △ | 取引サイトは審査厳しめ |
| SaaS 費用カバー | △ | 月 25,000 PV で可能だが、到達に時間 |
| LLM 費用カバー | × | 100,000 PV 以上必要、現実的でない |

## 提案

### オプション A: 広告実装を進める（段階的アプローチ）

**Phase 1（審査準備）:**

- プライバシーポリシー、利用規約ページ作成
- ads.txt 配置
- AdSense 審査申請

**Phase 2（実装）:**

- 広告コンポーネント作成（`src/components/ads/`）
- 非侵入型の広告配置（フッター上部のみ）
- 無料ユーザー判定ロジック

**Phase 3（最適化）:**

- A/B テストで配置最適化
- RPM モニタリング

### オプション B: 広告以外の収益化を優先

広告収益の限界を踏まえ、以下を優先する:

- **課金プラン（Pro/Premium）**: 広告非表示 + 追加機能
- **手数料モデル**: 取引成立時に少額手数料

### オプション C: 広告は見送り、コスト削減に注力

LLM 使用を最小化し、無料枠内で運営する。

## 推奨: オプション A + B のハイブリッドアプローチ

1. 広告は低コストで実装できるため、まず Phase 1-2 を進める
2. 同時に課金プランの設計を開始する
3. 広告だけで LLM 費用をカバーすることは期待しない
4. 課金プランが主収益源、広告は補助的位置づけ
